# Generated data

`packages.json` in this directory is a build-time artifact produced by
`scripts/sync-packages.ts`. It's committed as an empty `[]` placeholder so
`bun run dev` works on a fresh clone without first running the sync; the real
data is generated by:

```
bun run sync-packages
```

CI runs this before `astro build`, so production deploys always ship a fresh
catalog. Local dev that needs real data should run the same command.
